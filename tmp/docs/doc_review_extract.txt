[018] (normal) 1. Introduction
[019] (normal) This document describes the work completed during Sprint 1 of the Support Ticketing System (STS) project for the course we are taking.
[020] (normal) The objective of Sprint 1 was to implement the basic user access functionality of the system. During this sprint, we focused on building the initial foundation of the web application by implementing features related to user interaction with the system.
[021] (normal) The implemented functionality includes:
[022] (normal) Displaying the welcome screen when users visit the application
[023] (normal) Allowing anonymous users to register in the system
[024] (normal) Allowing registered users to log in
[025] (normal) Allowing logged-in users to log out
[026] (normal) These features form the core authentication and navigation structure of the application and serve as the basis for implementing additional functionality in later sprints.
[027] (normal) 2. System Architecture and Technologies
[028] (normal) In this sprint, we implemented the initial structure of the Support Ticketing System (STS) as a web-based client-server application. The system allows users to interact with the application through a web browser, which communicates with the server that processes requests and manages data.
[029] (normal) The client side of the system is the web browser, which displays the user interface and sends requests when users visit the homepage, register, log in, or log out. The server side is implemented using ASP.NET Core MVC, which handles the application logic and processes incoming HTTP requests.
[030] (normal) To store application data, we used a MySQL database. The server communicates with the database through Entity Framework Core, which allows the application to work with database data using C# objects.
[032] (normal) The main technologies used in this sprint include:
[033] (normal) ASP.NET Core MVC - for building the web application and handling requests
[034] (normal) C# - for implementing the application logic
[035] (normal) MySQL - for storing user data and other system information
[036] (normal) Entity Framework Core - for managing database communication
[037] (normal) ASP.NET Identity - for handling user authentication and password security
[038] (normal) Razor Views, HTML, SASS, Bootstrap - for creating the user interface
[039] (normal) This architecture allowed us to implement the required functionality for Sprint 1, including the welcome screen, user registration, login, and logout features.
